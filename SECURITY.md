# Security Policy

## Supported Versions

We currently provide security updates only for the latest version of this project.

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |

If you are using an older version, please update to the most recent commit.

---

## Reporting a Vulnerability

If you discover a **security vulnerability**, please follow responsible disclosure guidelines.

✅ **DO report privately**  
❌ **DO NOT open a public issue**

To report privately, contact the project maintainer:

- **Telegram:** @WTF_NoHope  
- **GitHub Security Advisory:** Create a private advisory in the "Security" tab.

We will review your report as soon as possible and work on a fix.

---

## What You Should Report

Please report issues like:

- API key / Token leaks  
- MongoDB connection exposure  
- Unauthorized access or privilege escalation  
- RCE (Remote Code Execution) risks  
- Sensitive information leaks  
- Broken authentication  
- Vulnerable dependencies  

---

## Response Time

| Severity | Expected Response |
|----------|-------------------|
| Critical | Within 24 hours   |
| High     | 48 hours          |
| Medium   | 3–5 days          |
| Low      | 1 week            |

We try to respond faster whenever possible.

---

## Responsible Disclosure

We appreciate your help in keeping AnanyaMusic secure.

Please:

✅ Give us time to fix issues before public disclosure  
✅ Provide clear and detailed reports  
✅ Avoid exploiting vulnerabilities in the wild  

You may receive credit for responsible disclosure (optional).

---

## Additional Security Notes

- **Do NOT commit `.env` files**  
- Rotate any leaked **bot tokens**, **API keys**, or **MongoDB URIs** immediately  
- Use strong passwords for all admin accounts  
- Avoid sharing sensitive logs publicly  
- Keep your local environment up to date  

Stay safe ❤️
